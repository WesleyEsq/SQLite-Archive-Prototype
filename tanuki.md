
# Tanuki Teacher | Trusted Archive Navigator

The purpose of **why I made this prototype** is to make a simple prototype for the automatic storage of large ammounts of metadata. I told this in the Readme.md file in this repository, but because it's an entirely different project, I am only telling here what I want out of this project.

---

## What are you doing, exactly?

I'm currently developing another project also using go, for a completely local AI Agent software using an LLM with a very low parameter count (the ballpark of 1 to 4 Billion parameters.) for the purpose of tutoring users with the use of documents as sources of information.

The hypothesis followed is that if a RAG database system is good enough, the AI will do way more than intended with some really high quality context fetched with the databases.

To achieve this task, I'm implementing a large polyglot persistence database setup, only using embeded databases because having to install any kind of database server for a simple AI tool isn't worth it for any kind of user. This isn't an AI project, and is almost entirely a database project.

---

## The why

We currently live in a world filled with cheap and quick Large language models for students, and this is taking a huge negative effect on their studies.

Because the easiest solution to make the threat of homework and projects go away, is to make a Gen AI do it for them in a few prompts rather than learn how to do it themselves in an afternoon.

This is detrimental for the intended path school programs define, from the elementary to the university level. The most vulnerable population being children, due to the ease of homework inteded at the elementary level, Ai could be close at solving any problem the kids could have.

Humanity are a people that try to choose the path of least resistance, and the current incentives for not using Gen AI are that it's not good enough in their job, or that they can't trust it with their asignments, but these limitations could only be temporary for most common use cases.

---

## The how

So I belive in response that a good path to take in the education sector would be to implement AI in simple systems that could help the learning process currently in curriculums. By accepting documents and other file formats, an AI Agent could help solve queries in the intended learning process. Acting as a personal tutor, similarly to NotebookLLM.

It cannot do an essay for you (because of the low parameter count), but it can explain questions about books and other study materials, so that the student can use the AI as a learning source.

I hope to develop a software that can be a personalized tutor, solve questions with large ammounts of files with context and respond in a LoRa Tuned way for students from elementary school to university and researchers individually with either Qwen3.5 or Ministral. (haven't decided yet).

I'm currently doing so at a local scale with high end consumer computers in mind, because of a couple of details regarding the current Gen AI LLM situation:

1. I simply don't have the funds to host an LLM in the long term for development.

2. Relying on Edge Computing can make the deployment of the application much more secure and feasible.

3. For thep purpose of research and development, there's an imposition on not using any form of external AI due to privacy concerns. So by offering a completely local software, this project could be allowed for an entire very important sector of local AI users.
